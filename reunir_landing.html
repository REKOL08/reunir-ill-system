Flujo del proceso

Cada etapa se gestiona automáticamente. Pasa el cursor sobre cada paso para más detalle.

📝

Usuario llena el formulario

El usuario completa el Google Form con los datos del material solicitado y su institución.
→
📨

Notificación a biblioteca prestadora

El sistema detecta la nueva solicitud y envía un correo automático a la biblioteca con botones Aprobar / Rechazar.
→
✅

Aprobación o rechazo

El bibliotecario responsable aprueba o rechaza con un clic desde su correo, sin necesidad de entrar al Sheet.
→
📖

Préstamo efectivo

El solicitante confirma la recepción del material. El sistema registra la fecha y calcula la fecha de devolución.
→
📅

Recordatorio automático

3 días antes del vencimiento, el sistema envía un recordatorio automático al solicitante.
→
📗

Devolución registrada

El solicitante confirma la devolución. El sistema cierra el ciclo y notifica a ambas partes.

Funcionalidades del sistema

🤖

Automatización total

Desde que el usuario envía el formulario hasta la devolución, el sistema gestiona cada etapa sin intervención manual.

📧

Notificaciones inteligentes

Correos HTML personalizados para cada actor del proceso: solicitante, biblioteca prestadora y administrador de la red.

🖱️

Aprobación con un clic

El bibliotecario aprueba o rechaza directamente desde su correo, sin necesidad de acceder a ningún sistema adicional.

📜

Trazabilidad completa

Registro cronológico de cada acción: quién actuó, qué cambió y cuándo — visible en la hoja de control.

⏰

Control de vencimientos

Recordatorios automáticos 3 días antes del vencimiento y marcado automático de préstamos vencidos cada mañana.

🏛️

Multi-institución

Diseñado para redes de bibliotecas con diferentes dominios de correo y responsables por institución.
