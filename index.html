El problema que resuelve

El proceso actual presenta limitaciones críticas para la operación de la red

❌ Antes — Estado actual

  • ⚠️ Google Forms sin automatización
  • ⚠️ Monitoreo manual de Excel
  • ⚠️ Sin notificaciones automáticas
  • ⚠️ Sin trazabilidad del estado
  • ⚠️ Sin recordatorios de devolución
  • ⚠️ Desarticulación entre bibliotecas

✅ Después — Sistema Reunir

  • ✔️ Flujo automatizado de extremo a extremo
  • ✔️ Notificaciones instantáneas por Gmail
  • ✔️ Aprobación con un clic desde el correo
  • ✔️ Historial cronológico por solicitud
  • ✔️ Recordatorios automáticos de vencimiento
  • ✔️ Evento en Calendar por cada préstamo

Flujo del proceso

Cada etapa se gestiona automáticamente. Pasa el cursor sobre cada paso.

📝

Usuario llena el formulario

El usuario completa el Google Form con los datos del material solicitado y su institución.
→
📨

Notificación a biblioteca prestadora

El sistema envía correo automático a la biblioteca con botones Aprobar / Rechazar.
→
✅

Aprobación o rechazo

El bibliotecario aprueba o rechaza con un clic desde su correo.
→
📖

Préstamo efectivo

El solicitante confirma la recepción. Se registra la fecha y calcula la devolución.
→
📅

Recordatorio automático

3 días antes del vencimiento se envía recordatorio automático al solicitante.
→
📗

Devolución registrada

El solicitante confirma la devolución y el sistema cierra el ciclo.

Funcionalidades del sistema

🤖

Automatización total

Desde el formulario hasta la devolución, el sistema gestiona cada etapa sin intervención manual.

📧

Notificaciones inteligentes

Correos HTML personalizados para cada actor: solicitante, biblioteca prestadora y administrador.

🖱️

Aprobación con un clic

El bibliotecario aprueba o rechaza directamente desde su correo, sin acceder a ningún sistema adicional.

📜

Trazabilidad completa

Registro cronológico de cada acción: quién actuó, qué cambió y cuándo — visible en la hoja de control.

⏰

Control de vencimientos

Recordatorios automáticos 3 días antes y marcado de préstamos vencidos cada mañana a las 8am.

🏛️

Multi-institución

Diseñado para redes con múltiples bibliotecas, diferentes dominios de correo y responsables por institución.
